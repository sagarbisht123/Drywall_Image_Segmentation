# 🧱 Prompted Segmentation for Drywall QA

Fine-tuning **CLIPSeg** for text-conditioned binary segmentation of drywall defects — given an image and a natural-language prompt, the model produces a binary mask for either cracks or taping areas.

---

## 🎯 Goal

Train a prompted segmentation model that accepts an image + text prompt and outputs a binary mask for:
- `"segment crack"` — wall cracks (Dataset 2)
- `"segment taping area"` — drywall joint/tape regions (Dataset 1)

## 🧠 Model

**CLIPSeg** (`CIDAS/clipseg-rd64-refined`) — a vision-language model that conditions image segmentation on free-form text prompts using CLIP embeddings.

- Parameters: **150.7M** (fully fine-tuned)
- Checkpoint size: **575 MB**
- Input image size: **352 × 352**

## 🖼️ Output Format

Prediction masks are saved as:
- **PNG**, single-channel, same spatial size as source image
- Pixel values: `{0, 255}`
- Filename format: `{image_id}__{prompt}.png`
  - e.g. `crack_abc123__segment_crack.png`

---

## ⚠️ Failure Notes

- **Taping area scores lower** (val mIoU ~0.50 vs ~0.74 for cracks) because Dataset 1 only has bounding box annotations — masks are rectangle approximations, not precise outlines of the tape
- The model occasionally over-segments crack regions on heavily textured walls where the background resembles crack patterns
- Very thin hairline cracks are sometimes missed due to low foreground pixel density
